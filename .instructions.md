# Project Instructions - Next.js 16 + Strapi 5 Full Stack

## 🌐 Language
**Always respond in Russian (Русский язык). All communication with the developer must be in Russian.**

## Project Stack
- **Frontend**: Next.js 16 (App Router)
- **Backend**: Strapi 5 (Headless CMS)
- **Styling**: Tailwind CSS v4+
- **UI Components**: shadcn/ui (built on Radix UI)
- **Database**: PostgreSQL
- **Caching**: Redis
- **Language**: TypeScript (strict mode)

## Code Style & Standards

### General
- Use TypeScript with strict mode enabled
- Write meaningful variable and function names
- Keep functions focused and under 50 lines
- Add JSDoc comments for public APIs and complex logic
- Use const-first approach

### Frontend (Next.js)
- Use App Router structure
- Implement Server Components by default
- Use Client Components sparingly with "use client" directive
- Place API routes in `app/api/` directory
- Use React hooks properly (useCallback, useMemo for optimization)
- Implement error boundaries for graceful error handling
- Use shadcn/ui components (built on Radix UI)
- Fulle SEO best practices (meta tags, semantic HTML, etc.)

### UI/Styling
- Use Tailwind CSS for styling
- Add components via shadcn/ui CLI, customize with Tailwind
- Follow mobile-first responsive design
- Maintain consistent spacing (use Tailwind scale)
- Create reusable component variants with CVA pattern

### Backend (Strapi 5)
- Use Strapi content types for data structure
- Configure API permissions via users-permissions plugin
- Use lifecycle hooks for custom logic (beforeCreate, afterUpdate, etc.)
- Create custom plugins for complex business logic
- Use Strapi ORM for database operations (auto-generated)
- Implement proper error handling in custom controllers
- Use middleware for request processing
- Configure CORS for Next.js integration

### Database (PostgreSQL)
- Strapi auto-generates database schema from content types
- Use Strapi relations (One-to-Many, Many-to-Many)
- Add indexes on frequently queried columns (via Strapi configuration)
- Use lifecycle hooks for complex data logic
- Write transactions for complex operations (in custom plugins)

### Caching (Redis)
- Use Redis for session management
- Cache expensive database queries
- Set appropriate TTL values
- Invalidate cache on data updates
- Use Redis for rate limiting

## Development Workflow

1. **Environment Setup**: Use `.env.local` for sensitive variables
2. **Database**: Run PostgreSQL in Docker, migrations on startup
3. **Redis**: Use local Redis for caching/sessions
4. **API Communication**: Use strongly typed API clients
5. **Testing**: Write tests for critical paths

## Performance Guidelines

- Implement pagination for large data sets
- Use database query optimization (relations, lazy loading)
- Cache frequently accessed data in Redis
- Monitor API response times
- Use Code splitting in Next.js for large bundles
- Optimize images with Next.js Image component

## Security

- Validate and sanitize all inputs
- Use HTTPS in production
- Implement CSRF protection
- Use secure session management (Redis + httpOnly cookies)
- Sanitize database queries (use parameterized queries)
- Never expose sensitive information in client code
- Implement rate limiting with Redis
- Use environment variables for secrets

## Commit Guidelines

- Use conventional commits (feat:, fix:, refactor:, etc.)
- Reference issue numbers when applicable
- Keep commits atomic and logical
